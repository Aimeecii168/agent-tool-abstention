Key Insight
The model made twice as many false negatives (34) as false positives (17) — meaning it is more likely to incorrectly execute a tool than to incorrectly abstain. In safety-critical deployment, a missed abstention can trigger a booking, a deletion, or an irreversible API call. Reducing FN is the priority.

Key Insight
Threshold is a deployment control, not a model parameter. Lowering it makes the system more conservative — the agent abstains more readily, reducing dangerous false negatives at the cost of more unnecessary abstentions. Raising it makes the system more action-oriented. The right threshold depends on the cost of each error type in your specific deployment environment.
