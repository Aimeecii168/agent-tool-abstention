Threshold as a deployment control
Think of the slider as a confidence dial. At 50%, the model only abstains when it's at least 50% confident it should. Drag it left to make the model more cautious — it abstains more often, catching more dangerous errors. Drag it right to make it more aggressive — it acts more often, but risks more irreversible mistakes. All results use actual predictions from our model — not simulated values.

Key Insight
At the default threshold of 0.50, we still have 30 dangerous errors. But at t=0.30, that drops to just 9. Yes, we get more unnecessary abstentions — but remember, that's the safe error. That's the tradeoff we want in a safety-critical system. t=0.30 with the deeper model is our recommendation.
Final Recommendation
Deploy the deeper Random Forest (max_depth=15) with threshold t=0.30, which achieves the best F1 among all tested configurations (90.95%) and the lowest false negatives among tested thresholds (9). In safety-sensitive tool environments — where incorrect tool execution can trigger irreversible actions — the additional false positives (33 unnecessary abstentions) are an acceptable cost for dramatically reducing dangerous missed abstentions. Threshold should be revisited if the cost of unnecessary abstentions becomes measurable in production.
